# 7페이지+ / 11페이지 페이지 이동 실패 원인 분석

## 현상
1. **7페이지부터**: 페이지 이동이 오래 걸리고 실패 빈도 증가
2. **11페이지**: 두 번 "다음" 클릭이 필요한데, 육안으로 11페이지 이동 확인했으나 프로그램은 인식하지 못하고 목록 리로드 후 재시도 → 종료

## 근본 원인

### 1. 블록 페이징 구조
국가법령센터는 **5페이지 단위 블록**으로 페이징됩니다.
- 블록 0: 1~5
- 블록 1: 6~10  
- 블록 2: 11~15
- ...

6페이지 이상으로 가려면 **"다음"(→) 버튼**을 눌러 블록을 전환한 뒤 해당 페이지 번호를 클릭해야 합니다.
11페이지는 블록 2의 첫 페이지이므로 **다음 버튼 2번** 클릭이 필요합니다.

### 2. 타이밍 부족
- `need_next >= 2` (11페이지 등): next 2번 클릭 후 3초만 대기 → 11~15 블록 DOM이 완전히 렌더링되기 전에 linkPage(11) 호출
- linkPage/클릭 후 `time.sleep(3)` → AJAX 목록 갱신이 3초 내에 끝나지 않을 수 있음
- `_wait_for_list_ready`가 프레임 구조에서 `precSeq=` 검사에 실패해 거의 항상 타임아웃 → 실제로는 고정 대기에 의존

### 3. 블록 첫 페이지 특수 케이스
- 현재 `page_num == 6`만 "next 후 이미 해당 페이지일 수 있음"으로 처리
- 11, 16, 21...도 동일 패턴: next N번 후 해당 블록 첫 페이지가 자동 로드될 수 있음

### 4. linkPage 호출 컨텍스트
- 주석: "linkPage(n)을 부모 윈도우에서 호출 (프레임 내부 아님)"
- 현재 content 프레임에서 호출 중 → 부모/탑 윈도우에서 먼저 시도하는 것이 안정적일 수 있음

### 5. 리로드 시 from_block_page=0
- 클릭 실패 → pageIndex 실패 → 리로드 후 `from_block_page=0`
- 11페이지: `need_next=2` (next 2번) → next 2번 후 대기 시간이 핵심
